About

Welcome to the TPF Profiler demonstration page. The TPF Profiler allows for assessing the performance of Triple Pattern Fragment (TPF) server. There are four major steps to this process:

  1. Retrieve a sample of triple patterns from the TPF server
  2. Generate triple patterns based on the sample
  3. Measure the response time for all triple patterns
  4. Analyze the response time data
The goal of this demo is to present the TPF Profiler by providing a guided demo of the profiling process and allow user to try out the tool.

Getting started

In the following, you can find out how to use this demo. In principal, there are two major option to choose from: You can either analyze the provided examples or you can run the profiler and analyze the created results.

  1. Run the Profiler

    2. To run the profiler, simply go to the Run Profiler page and select the desired settings. You can choose from a selection of different Knowledge Graphs (KGs) made available via various TPF servers. For this demo, we provide a locally deployed TPF server with four KGs and a remote publicly available TPF server hosted by the Linked Data Fragments team. Thereafter, the sample size of can be configured (for performance reasons, we allow for a maximum number of 100 samples). Once the profiler is configured, you can start it by hitting the "Run Profiler" button. You will be redirected to a page which allows for monitoring the current status of the profiler. Please note, that the profiling process might take a few minutes depending on the selected server and sample size.

  2. Visualize/Download the results

    3. When the profiling process is finished, you can either download the data created by the profiler (as a CSV file) or you can use the preselected visualizations of the demo to analyze the data. The results of the profiling process are stored until 10pm (CEST). All available results are listed on the List Results page. On this page, we have also provided a set of examples which are always available for analysis.

  3. Analyze and compare the results

    4. In this demo, we provide a selection of visualizations to analyze both the repsonse time of the results as well as the samples and triple patterns created by the profiler and used for the performance measurment. The visulizations can be accessed directly after the profiling process is completed using the "Visualize Results" button or via the List Results page on which all available results are provided. Furthermore, the demo allows for an A/B comparision of two selected results on a single page, which facilitates the comparison of the available results. The comparisons are available via the Compare Results page. On this page two dropdown menus allow for selecting the desired results from all available results.
Please note that running the profiler and loading the results for visualization might take some time.
